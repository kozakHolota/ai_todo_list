# AI Todo List — Розумний менеджер завдань

Інтелектуальна система управління завданнями на базі штучного інтелекту. Замість традиційних кнопок і форм — проста розмова з AI-агентом. Ви пишете звичною мовою, агент розуміє і виконує потрібну дію.

## Можливості

| Операція | Опис |
|---|---|
| Додавання завдання | Створення нового завдання з назвою та необов'язковою датою виконання |
| Перегляд списку | Відображення всіх завдань із поточним статусом |
| Позначення виконаним | Зміна статусу завдання на COMPLETED |
| Редагування | Оновлення назви, опису, дати або пріоритету |
| Видалення | Повне видалення завдання зі списку |
| Пошук | Пошук за ключовим словом, статусом або часовим діапазоном |
| Деталі завдання | Перегляд усіх полів конкретного завдання |

Агент підтримує **відносні дати**: «завтра», «через 2 години», «у п'ятницю» тощо.

## Запуск

1. Відкрийте файл `todo_agent.ipynb` у Jupyter:

```bash
uv run jupyter notebook todo_agent.ipynb
# або
jupyter notebook todo_agent.ipynb
```

2. Виконайте першу клітинку — вона запитає ваш OpenAI API-ключ.

3. Оберіть режим роботи:

**Автоматичний тест** — запускає заздалегідь підготовлені приклади, щоб перевірити роботу агента.

**Інтерактивний режим** — ви самостійно вводите запити і спілкуєтесь з агентом.

## Як користуватись

### Основний принцип

Пишіть агенту звичними реченнями. Немає потреби вивчати команди чи синтаксис — агент сам визначить потрібну операцію.

### Статуси завдань

| Статус | Значення |
|---|---|
| `PENDING` | Очікує виконання (за замовчуванням) |
| `IN_PROGRESS` | Виконується |
| `COMPLETED` | Виконано |
| `FAILED` | Завершилось невдачею |
| `CANCELED` | Скасовано |

## Приклади запитів

### Додавання завдань

```
Додай завдання: зателефонувати лікарю
```
```
Додай: купити продукти — до завтра о 18:00
```
```
Нагадай мені написати звіт через 3 години
```

### Перегляд списку

```
Покажи всі завдання
```
```
Що у мене заплановано?
```
```
Які завдання ще не виконані?
```

### Позначення виконаним

```
Позначи "зателефонувати лікарю" як виконане
```
```
Завдання про звіт виконано
```

### Редагування

```
Перенеси дедлайн завдання про продукти на п'ятницю
```
```
Змін назву завдання "купити продукти" на "купити продукти та ліки"
```

### Видалення

```
Видали завдання про звіт
```
```
Прибери всі виконані завдання
```

### Пошук та деталі

```
Знайди завдання пов'язані з лікарем
```
```
Покажи деталі завдання про продукти
```
```
Які завдання заплановані на сьогодні?
```

## Структура проєкту

```
ai_todo_list/
├── todo_agent.ipynb    # Головний файл застосунку (Jupyter notebook)
├── llm.py              # Ініціалізація AI-агента та LLM
├── models/
│   ├── response.py     # Модель даних Task (Pydantic)
│   └── state.py        # Перелік статусів TaskState
├── tools/
│   ├── task_tools.py   # Реалізація всіх інструментів агента
│   └── util.py         # Допоміжні функції
├── pyproject.toml      # Метадані проєкту та залежності
└── requirements.txt    # Список залежностей
```

### Ключові компоненти

- **LangGraph ReAct агент** — агент розмірковує над запитом і вирішує, який інструмент використати
- **LangChain tools** — кожна операція (додати, видалити, оновити тощо) реалізована як окремий інструмент
- **Pydantic моделі** — валідація та структурування даних завдань
- **InMemoryStore** — зберігання завдань у пам'яті під час сесії

## Обмеження

- **Дані не зберігаються між сесіями.** Після перезапуску Jupyter-ядра всі завдання втрачаються.
- Для роботи потрібен активний OpenAI API-ключ з доступом до GPT-4.
- Агент не підтримує роботу в кількох паралельних сесіях одночасно.
